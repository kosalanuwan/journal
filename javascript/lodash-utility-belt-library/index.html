Writing a Lodash-like JS utility belt library

5th November 2016
2 minute read
Tweet

I prefer having my own JS utility belt library like they do it on Lodash and Underscore.js, say, a stripped-down version of it, then bundled together into global namespace as _.

Example 1

Use as typical _ utility functions

_.format('Hello {0}!', username);
_.kebabCase('Some teXt to Kebab case');
_.isblank(username);

The first thing we’re going to need is a few utility libraries. Fortunately, GitHub has plenty of opensourced utility libraries. Lodash, Underscore.js, Angular.js code arn’t difficult to understand as Babel code. So, I can run a search in the repo and it’ll return all of the code snippets and function definitions. Then I can break out the scripts in individual JS files that are in and save them as my own utility belt library. All I need is a little polyfill magic with a namespace, say, _ and we’re set. Moving right along.

Example 2

Do the same with mixin and prototype option

'Hello {0}!'.format(username);
'Some teXt to Kebab case'.toKebabCase();
username.isblank();

Next is Mixin. You can run all the scripts in a namespace and is slightly obnoxious if not available as part of the browser API or native JS object too. You’d be amazed how intense it is to enable both namespace and prototype options. Not only this require a Closure pattern, but you also have to emulate methods. @esamattis/underscore.string has mixin for String types. So I’m gonna go ahead and break out that same script and modify my utility script. Excellent.
